# 飞机机制说明

本文档介绍 `unit` package 中飞机的主要运行时机制，重点说明舰载机从出动、起飞、巡航交战到返航着舰的完整流程。

这里描述的是当前代码行为。飞行阶段与着舰轨迹均为局内运行时逻辑，不增加 `planes.json5` 或舰船配置字段。

## 代码分工

| 文件 | 职责 |

| `aircraft.go` | 航母机库、起飞冷却、库存扣减、着舰槽位和飞机回收 |
| `movement_strategy.go` | 巡航 / 交战阶段的普通移动和战斗机追踪 |
| `flight_phase.go` | 飞行阶段定义、起飞流程、视觉倍率及通用插值函数 |
| `landing_phase.go` | 降落阶段的初始化、状态推进、速度控制和动态时长 |

## 生命周期

飞机的公开飞行阶段由 `PlaneFlightPhase` 表示：

```text
taking_off
    |
    v
cruising ---------------> landing_staging
                               |
                               v
                        landing_approach
                               |
                               v
                          landing_deck
                               |
                               v
                     恢复库存并移除活动飞机
```

- `taking_off`：沿航母航向完成甲板滑跑和加速。
- `cruising`：正常巡航、接敌、追踪和开火。
- `landing_staging`：捕获舰尾远端引导点，再沿入口切线进入圆弧起点。
- `landing_approach`：沿单向定半径圆弧汇入航母中线。
- `landing_deck`：沿中线完成最后 `1.5` 个舰长的减速、缩放和甲板回收。

`PlaneReturn.Exec` 只负责上述状态的切换和最终回收。具体的速度与几何计算由 `flight_phase.go`、`landing_phase.go` 和 `landing_geometry.go` 负责。

飞机只有处于 `cruising` 时才会获得新的攻击行为并主动开火。起飞或降落中的飞机仍保留在对局对象集合中，因此仍可受到已有防空伤害检查的影响。

## 出动与巡航

### 出动选择

`ShipAircraft.TakeOff` 根据目标对象类型选择机组：

- 战斗机攻击飞机。
- 俯冲轰炸机和鱼雷轰炸机攻击舰船。
- 没有机枪、炸弹、鱼雷或火箭的飞机不会被自动派出攻击。

起飞前会检查 `TakeOffTime` 冷却和对应机组的 `CurCount`。飞机生成时先扣减库存，再进入 `taking_off`。

### 巡航与交战

进入 `cruising` 后，飞机恢复现有移动和武器逻辑：

- 普通移动策略按飞机最大速度飞向引导目标。
- 战斗机移动策略会根据敌机位置和速度调整追踪距离与自身速度。
- 对舰飞机释放炸弹或鱼雷后执行一击脱离，由任务管理器决定下一目标或返航。
- `updatePlaneWeaponFire` 只遍历处于巡航阶段的飞机执行主动开火。

### 自动返航

`Plane.MustReturn` 使用以下条件：

- 任意飞机的 `RemainRange <= 0` 时必须返航。
- 俯冲轰炸机和鱼雷轰炸机的炸弹、鱼雷及火箭全部耗尽后返航。
- 战斗机目前主要在航程耗尽后返航。

如果所属航母已经不存在，返航飞机当前会被判定为坠毁；暂不支持转降其他航母。

## 起飞设计

### 滑跑位置与航向

飞机从航母中心生成，而不是从舰艏生成。初始航向与航母当前航向一致，随后沿该方向至少直飞 `2.0 * 舰长`，形成完整的甲板滑跑和离舰过程。

飞机需要同时满足两个条件才会切换到巡航：

1. 已飞完最短滑跑距离。
2. 已完成 `45` 个模拟帧的加速过程。

因此，短舰体或高倍速不会导致飞机在加速尚未结束时突然转向接敌。

### 加速曲线

初始滑跑速度为以下两者的较大值：

```text
航母当前速度
飞机最大速度 * 游戏速度倍率 * 15%
```

之后使用 `smoothstep` S 曲线在 `45` 个模拟帧内插值到飞机最大速度：

```text
smoothstep(t) = t^2 * (3 - 2t),  t in [0, 1]
```

曲线在首尾的一阶导数为 `0`，可避免起步和完成起飞时出现单帧速度跳变。游戏速度倍率同时影响阶段计时和位移，因此只改变播放节奏，不改变轨迹形状。

### 起飞视觉倍率

常规飞机图片按舰船比例的约 `2` 倍绘制。起飞阶段的 `VisualScaleMultiplier` 从 `0.5` 平滑增加到 `1.0`，对应最终视觉效果：

```text
甲板低空：常规飞机比例 2x * 0.5 = 舰船同级 1x
巡航高度：常规飞机比例 2x * 1.0 = 当前飞机 2x
```

缩放仅影响图片绘制，不改变命中、碰撞或战斗数值。

## 降落设计

降落轨迹分为三段：切线引导、定半径圆弧和直线着舰。核心目标是让飞机以单一转弯方向汇入中线，避免从舰侧强行折转、S 形摆动或在舰尾形成锐角。

### 航母局部坐标

所有着舰几何先在航母局部坐标中计算，再转换到地图坐标：

```text
                    forward +（舰艏）
                           ^
                           |
 lateral -（左舷） <--- 航母中心 ---> lateral +（右舷）
                           |
                           v
                    forward -（舰尾）
```

- `forward` 朝舰艏为正。
- `lateral` 朝舰体右舷为正。
- 圆弧参数以舰长为单位，转换为地图位置或实际速度时再乘以航母地图长度。

轨迹绑定航母的实时位置与航向。航母平移或转向时，进近点不会留在旧的世界坐标中。

### 槽位与入口分散

飞机开始返航时，`ShipAircraft.RequestLanding` 为其分配一个稳定槽位。槽位使用当前最低的空闲编号，飞机回收或被移除后释放，后续飞机可以复用。

每组共有 `16` 条左右交错的并行入口：

- 基础入口位于航母中心后方约 `3.5` 个舰长。
- 横向距离从 `0.8` 到 `1.4` 个舰长。
- 超过 `16` 架后进入下一组，每组再向舰尾错开 `0.12` 个舰长。

这些槽位用于分散同时返航的飞机，不是只允许一架通过的 FIFO 单通道队列。实现也不进行飞机间碰撞、两两避让或寻路。

### 切线引导

`landing_staging` 内部包含两个步骤，但不会增加额外的公开飞行阶段：

1. `lead-in`：飞向圆弧入口切线后方 `1.5` 个舰长的远端引导点。
2. `gate`：沿入口切线捕获圆弧起点，并提前调整到圆弧所需速度。

只有同时满足以下条件，飞机才会进入定半径圆弧：

- 纵向误差不超过 `0.25` 个舰长。
- 横向误差不超过 `0.20` 个舰长。
- 航向误差不超过 `8` 度。
- 速度相对误差不超过约 `10%`，或不超过一个正常单帧变速步长。

如果飞机越过入口或穿过航母中线，会回到远端引导点重新建立切线，不会从舰侧或舰前强行生成一条进近曲线。

入口附近使用切线速度场控制飞机：纵向速度保持向入口推进，横向修正与离切线的距离成比例，并限制在圆弧相对速度的 `25%` 以内。该限制可以逐渐消除横向误差，同时避免过度修正造成左右摆动。

远距离返航允许飞机提高速度追赶移动航母；进入距航母约 `6` 个舰长的范围后开始减速，在约 `4` 个舰长内收敛到圆弧入口速度。速度包络按飞机到航母的径向距离计算，不会因为切换引导点而突然重新加速。

### 定半径圆弧

`landing_approach` 使用单向定半径圆弧，不再使用可能产生反曲的单段 Bézier 曲线。

圆弧从飞机实际进入 gate 时的位置开始，终点固定为航母中线后方 `1.5` 个舰长。终点切线与航母航向完全一致，便于无折角地进入最终直线段。

设入口到终点的纵向距离为 `dx`，入口横向距离的绝对值为 `y`。为使圆同时经过入口和终点，并在终点与中线相切，半径为：

```text
R = (dx^2 + y^2) / (2 * |y|)
```

该构造具有以下性质：

- 左右入口生成互为镜像的航线。
- 整段保持同一转弯方向和固定曲率。
- 纵向位置持续朝甲板推进。
- 横向偏差持续收敛且不穿越中线。
- 圆弧出口航向与最终直线着舰方向连续。

圆弧时长按弧长和机型速度计算，并限制在 `90-120` 个模拟帧。飞机进入圆弧前已经在切线引导段匹配入口速度，因此阶段切换不会触发瞬间加速。

### 移动和转向中的航母

圆弧上的飞机世界速度由三部分组成：

```text
飞机世界速度 = 航母平移速度
             + 飞机相对航母的圆弧切向速度
             + 航母旋转产生的 omega x r 速度
```

飞机图片朝向取该世界速度的方向，而不是只取圆弧的局部切线。这样航母持续移动或转向时，飞机的位置、朝向和速度仍与绑定在航母上的轨迹一致。

### 最终直线着舰

`landing_deck` 从舰尾后 `1.5` 个舰长处开始，沿航母中线移动到甲板中心。位置使用二次 ease-out：

```text
easeOut(t) = 1 - (1 - t)^2
```

它的导数从最大值线性下降到 `0`，因此飞机相对航母的前向速度会持续降低，并在甲板中心与航母速度一致。

直线段时长根据圆弧出口相对速度动态计算：

```text
frames = 2 * distance / relativeSpeed
```

其中系数 `2` 来自二次 ease-out 在起点的导数。结果限制在 `60-180` 个模拟帧，以保持不同舰长和机型下的节奏稳定，同时尽量保证圆弧末帧与直线首帧速度连续。

视觉倍率在该阶段使用 `smoothstep` 从 `1.0` 降到 `0.5`，即图片从当前飞机的 `2x` 逐渐缩小到舰船同级的 `1x`。

### 回收时机

飞机只有完成直线着舰并到达甲板中心后，`PlaneReturn` 才会调用 `ShipAircraft.Recovery`：

- 释放该飞机的降落槽位。
- 生命值不低于总生命值 `15%` 时，恢复对应机组一架库存。
- 生命值低于 `15%` 时视为无回收价值，不恢复库存。
- 无论是否恢复库存，已着舰飞机都会从活动飞机集合删除。

当前回收后仍沿用原有起飞冷却逻辑，本文档不涉及整备、维修或重新出动队列。

## 性能约束

起降逻辑位于游戏更新热路径，当前实现遵循以下限制：

- 每架飞机每帧只进行常数次坐标变换、插值和三角函数计算。
- 不进行返航飞机之间的两两距离检查。
- 不进行碰撞避让、动态寻路或轨迹搜索。
- 总复杂度为 `O(活动飞机数)`，而不是 `O(n^2)`。
- 圆弧采样和阶段更新不在热路径产生堆分配，相关测试包含零分配断言。

为了让地图边缘、舰艏朝外的航母仍能建立舰尾进近航线，`landing_staging` 允许飞机短暂位于常规地图边界外；进入圆弧和甲板段后仍按绑定航母的轨迹完成回收。

## 关键参数

以下参数目前均为代码内常量，不属于配置项：

| 参数 | 当前值 | 含义 |
| --- | ---: | --- |
| `carrierTakeoffLaneRatio` | `2.0` | 最短起飞滑跑距离，单位为舰长 |
| `takeoffInitialSpeedRatio` | `0.15` | 静止航母上的初始滑跑速度比例 |
| `takeoffAccelerationFrames` | `45` | 起飞 S 曲线加速时长 |
| `landingStagingBaseForwardRatio` | `-3.5` | 第一组圆弧入口的纵向位置 |
| `landingApproachLanes` | `16` | 每组左右交错入口数量 |
| `landingLaneMinLateralRatio` | `0.8` | 最内侧入口横向距离 |
| `landingLaneMaxLateralRatio` | `1.4` | 最外侧入口横向距离 |
| `landingLeadInLengthRatio` | `1.5` | 入口切线远端引导长度 |
| `carrierLandingFinalStartRatio` | `-1.5` | 最终直线着舰起点 |
| `landingApproachMinFrames` | `90` | 圆弧最短时长 |
| `landingApproachMaxFrames` | `120` | 圆弧最长时长 |
| `landingDeckMinFrames` | `60` | 最终直线段最短时长 |
| `landingDeckMaxFrames` | `180` | 最终直线段最长时长 |
| `planeLowAltitudeVisualScale` | `0.5` | 相对常规飞机绘制比例的低空倍率 |

调整这些数值时，应同步检查位置、航向、速度和视觉倍率的连续性，不能只观察单个阶段的耗时。

## 测试覆盖

相关测试主要位于：

- `flight_phase_test.go`：起飞距离、速度单调性、视觉倍率和阶段切换。
- `landing_phase_test.go`：槽位分散、入口重试、圆弧镜像与曲率、移动 / 转向航母、地图边界、速度连续性、批量回收及零分配。
- `../../instruction/plane_test.go`：返航指令流转，以及飞机只有经过甲板中心后才恢复库存并移除。

修改起降逻辑后，至少运行：

```bash
go test ./pkg/mission/object/unit ./pkg/mission/instruction
go test ./...
```

轨迹自然度、图片缩放和多机同时返航仍属于视觉行为，测试通过后还应启动游戏进行较长时间的画面冒烟验证。
