# 战力计算说明

`combatpower` 包根据完成初始化的飞机、舰船、武器和弹药对象计算静态战力。结果用于游戏图鉴和数值平衡分析，不会修改实际伤害、命中、AI、费用或增援时间。

## 计算时机

初始化顺序为：

1. 加载弹药和各类武器，并完成运行时单位换算。
2. 加载飞机和舰船，并解析其武器引用。
3. 先调用 `CalculatePlane` 计算全部飞机。
4. 再调用 `CalculateShip`，使航母可以读取舰载机战力。
5. 最后加载图鉴引用数据。

`Plane.CombatPower` 和 `BattleShip.CombatPower` 使用 `json:"-"`，不会从 JSON5 读取，也不会形成第二份需要维护的战力配置。

## 输出字段

| 字段           | 含义                             |
|--------------|--------------------------------|
| `FormationSize` | 本条战力代表的单位数量；舰船为 1，飞机为 10       |
| `Total`      | 综合战力；普通单位按对舰、对空加权，航母为舰体与航空战力之和 |
| `AntiShip`   | 对舰战力                           |
| `AntiAir`    | 对空战力                           |
| `Survival`   | 生存能力值                          |
| `Mobility`   | 机动能力值                          |
| `Projection` | 投送能力值，供雷达图比较，不是公里数             |
| `Burst`      | 一轮齐射或齐投的爆发能力值                  |
| `Hull`       | 舰船本体综合战力                       |
| `Aviation`   | 航母满编舰载机的折算战力                   |
| `Details`    | EHP、有效 DPS、投送距离、爆发伤害及各武器贡献明细   |

所有最终能力值均四舍五入为非负整数。非法值、`NaN` 和无穷值会被压回 `0`。

## 伤害与输出

### 单发期望伤害

```text
单发期望伤害 = Damage × (1 + 2.7 × CriticalRate)
```

这里的 `2.7` 是暴击额外收益系数。弹药不存在、伤害不为正时，期望伤害为 `0`。

### 原始 DPS

火炮：

```text
DPS = 炮弹数 × 单发期望伤害 / ReloadTime
```

鱼雷发射器：

```text
发射周期 = ReloadTime + (炮弹数 - 1) × ShotInterval
DPS = 炮弹数 × 单发期望伤害 / 发射周期
```

舰载分组火箭：

```text
组数上限 = min(max(1, GroupCount), RocketCount)
每组数量 = ceil(RocketCount / 组数上限)
实际组数 = ceil(RocketCount / 每组数量)

发射周期 = ReloadTime
         + (RocketCount - 实际组数) × ShotInterval
         + (实际组数 - 1) × GroupInterval

DPS = RocketCount × 单发期望伤害 / 发射周期
```

飞机炸弹、鱼雷和火箭属于有限挂载，统一按 `60 秒`观察窗口折算：

```text
炸弹或鱼雷 DPS = 单次释放期望伤害 / 60
飞机火箭 DPS   = RocketCount × 单发期望伤害 / 60
```

同一飞机上的多个挂点会分别计算后累加。

### 有效输出系数

原始 DPS 和爆发伤害都会乘以相同的有效输出系数：

```text
有效输出系数 = 基础命中率 × 散布系数 × 射界系数 × 射程系数
有效 DPS      = 原始 DPS × 有效输出系数
```

基础命中率：

| 武器与目标   |     系数 |
|---------|-------:|
| 舰炮对舰    | `0.60` |
| 舰船防空炮对空 | `0.11` |
| 飞机航炮对空  | `0.55` |
| 鱼雷对舰    | `0.35` |
| 炸弹对舰    | `0.55` |
| 火箭对舰    | `0.45` |
| 火箭对空    | `0.50` |

飞机航炮在允许对舰时沿用炮击对舰系数 `0.60`。战斗机只保留对空统计；其他机种按照武器自身的 `antiShip`、`antiAircraft` 标记同时保留对舰和对空能力。该规则只作用于图鉴战力，不改变实战目标选择、AI 或伤害逻辑。

散布系数：

```text
散布系数 = clamp(ref / (ref + spread), 0.35, 1)
```

- 对舰 `ref = 100`
- 对空 `ref = 20`

射界系数：

```text
射界系数 = sqrt(左右射界并集角度 / 360)
```

左右射界的重叠部分只计算一次。

射程系数：

```text
射程系数 = clamp((weaponRange / ref)^0.15, 0.8, 1.25)
```

- 对舰运行时基准 `ref = 10`
- 对空运行时基准 `ref = 3`

这里的 `weaponRange` 是初始化后的运行时距离，只用于计算相对收益，不直接作为公里数展示。

## 生存能力

舰船 EHP 同时考虑水平和垂直减伤，并按 `6:4` 加权：

```text
水平 EHP = HP / max(0.001, 1 - 水平减伤)
垂直 EHP = HP / max(0.001, 1 - 垂直减伤)
舰船 EHP = 0.6 × 水平 EHP + 0.4 × 垂直 EHP
```

飞机单机 EHP 按三倍受伤系数折算：

```text
飞机 EHP = HP / (3 × max(0.001, 1 - 减伤))
```

`0.001` 下限用于保证 `100%` 减伤的特殊单位仍能得到有限结果。

舰船直接使用单舰 EHP；飞机图鉴使用 10 架标准编队，所有可累加的原始能力均扩大 10 倍：

```text
编队 EHP = 单机 EHP × 10
编队有效 DPS = 单机有效 DPS × 10
编队爆发伤害 = 单机爆发伤害 × 10
```

生存能力值为：

```text
Survival = round(sqrt(EHP))
```

## 机动能力

机动系数使用乘法模型，使速度、转向、加速度和航程分别提供较平缓的收益。

舰船：

```text
M = (MaxSpeed / 0.05)^0.12
  × (RotateSpeed / 2)^0.08
  × (Acceleration / 0.0005)^0.04

M = clamp(M, 0.75, 1.25)
```

飞机：

```text
M = (MaxSpeed / (500 / 5400))^0.12
  × (RotateSpeed / 12)^0.08
  × (Acceleration / 0.05)^0.03
  × (Range / (1500 / 14.4))^0.05

M = clamp(M, 0.75, 1.30)
```

以上参数均为初始化后的运行时值。机动能力值为：

```text
Mobility = round(100 × M)
```

## 对舰、对空与综合战力

对舰和对空分别使用对应口径的 EHP 与有效 DPS：

```text
战力 = round(sqrt(EHP × 有效 DPS) × 机动系数 / 10)
```

没有 EHP、有效 DPS 或机动系数时，该项战力为 `0`。

飞机使用 10 架编队后，EHP 与 DPS 同时乘以 10，因此对舰、对空战力约为未取整单机分数的 10 倍；这样可以避免单机小数过早取整后集中显示为 `0` 或 `1`。投送和机动属于单机的强度属性，不随编队数量放大。

普通单位的综合战力为：

```text
Total = round(0.7 × AntiShip + 0.3 × AntiAir)
```

若对舰或对空至少一项大于 `0`，但加权取整后得到 `0`，综合战力最低记为 `1`。

## 投送能力与公里换算

投送能力的原始计算使用初始化后的运行时距离：

```text
Projection = round(MaxProjectionRange × 10)
```

`Projection` 是雷达图使用的相对能力值，不是距离单位。图鉴 tooltip 使用 `MaxProjectionDistanceKM` 显示实际公里数。

舰船武器配置按现实公里数填写，初始化时统一执行 `Range /= 2`，因此展示时需要反向换算：

```text
舰船最大投送距离（km） = 运行时武器距离 × 2
```

飞机配置的航程在初始化时执行 `Range /= (8 × 1.8)`，即除以 `14.4`：

```text
飞机作战半径（km） = 运行时飞机航程 × 14.4
```

例如衣阿华主炮配置射程为 `38`：

```text
运行时距离 = 38 / 2 = 19
投送能力值 = 19 × 10 = 190
图鉴展示距离 = 19 × 2 = 38 km
```

航母分别计算舰体武器和舰载机的公里距离，取其中较大值作为最大投送距离，不能将舰载机航程错误地乘以舰炮系数 `2`。

## 爆发能力

单轮爆发为该武器一次齐射或完整挂载的期望伤害，再乘有效输出系数。飞机标准编队和不含航空贡献的舰体取对舰、对空爆发中较高的一侧：

```text
BurstDamage = max(对舰爆发伤害, 对空爆发伤害)
Burst = round(sqrt(BurstDamage))
```

爆发能力与持续 DPS 分开保存，不参与 `AntiShip`、`AntiAir` 或 `Total` 的计算。航母会按实际载机数折算标准编队爆发，再累加到舰体爆发。

## 航母航空贡献

航母按满编数量折算每个 10 架标准编队，并应用 `0.7` 航空折扣：

```text
编组系数 = 编组满编数量 / FormationSize × 0.7
航空对舰 = round(Σ(编组系数 × 飞机对舰战力))
航空对空 = round(Σ(编组系数 × 飞机对空战力))
Aviation = round(0.7 × 航空对舰 + 0.3 × 航空对空)

航母 Total = Hull + Aviation
```

航空明细中的 DPS、爆发伤害和武器贡献同样乘以编组系数。例如 5、10、20 架飞机分别按 0.5、1、2 个标准编队计算，再应用 `0.7` 折扣。投送距离取舰体武器与全部舰载机航程的最大值，不按飞机数量累加。

## 雷达图归一化

P95 归一化属于 `pkg/collection` 的展示逻辑，不会改变本包计算出的原始能力值：

- 舰船按当前“国籍 + 舰种”筛选结果建立比较池。
- 飞机按当前“国籍 + 机种”筛选结果建立比较池。
- 每个维度使用比较池的第 95 百分位作为满轴，超过该值时截断为 `100%`。
- 舰船和飞机使用独立尺度，避免直接混合舰炮与飞机能力。

## 调整系数时

修改计算模型时，应同步检查：

1. `combatpower.go` 中的公式、常量和目标限制。
2. `combatpower_test.go` 中的期望值、边界值和单位换算测试。
3. 本 README 中对应公式与系数。
4. `pkg/collection` 中雷达维度和 tooltip 是否仍能正确解释结果。
