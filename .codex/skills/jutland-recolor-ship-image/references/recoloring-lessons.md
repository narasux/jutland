# 战舰配色修改经验

## 1. 参考图只负责颜色

图像生成工具能快速给出“银白舰体加木甲板”的视觉方向，但可能改变画布尺寸、舰体轮廓、武器数量、文字和像素细节。需要进入游戏资源链的图片必须以原始目标图为几何真值，使用确定性像素处理完成最终版本。

## 2. 颜色不等于材质

原始战舰图常用有限调色板。同一个深蓝色可能同时表示：

- 木甲板底色或板缝；
- 主炮塔和副炮；
- 防空炮与炮座；
- 防鱼雷突出部；
- 舰体阴影。

因此不能直接全图执行一种颜色到另一种颜色的替换。至少结合以下信息判断：

- 位置：甲板内部、舰体外缘或上层建筑；
- 形状：长带、大块面、圆形炮座或小型重复组件；
- 连通面积与包围盒；
- 左右或上下对称性；
- 参考舰相同部位的材质。

## 3. 新墨西哥号参考调色板

以下颜色来自本次参考图，可作为相近 Shipbucket 图片的起点，但每次仍应从用户给定的参考图重新取样：

| 材质 | 常见 RGB | 用途 |
|---|---:|---|
| 木甲板浅色底 | `239,228,176` | 蜂蜜色、黄蜡色甲板底 |
| 木甲板板缝/深色层 | `217,164,91` | 木纹、板缝和局部暖色阴影 |
| 浅灰金属 | `213,213,213` | 高光、浅色上层建筑 |
| 主金属灰 | `193,193,193` | 舰体、炮塔、防雷突出部 |
| 中灰金属 | `174,174,174` | 金属阴影与设备层次 |
| 水下舰体红褐 | `134,58,51` | 默认保留，不随水上涂装修改 |

银白涂装不是单一纯白。至少保留浅灰、主灰和中灰层次，否则线稿会显得扁平，白色区域也容易与透明背景混淆。

## 4. 木甲板识别

木甲板通常具有：

- 覆盖范围大且位于俯视舰体内部；
- 大量规律的横向或纵向板缝；
- 暖黄底色与较深橙黄色纹理同时出现；
- 被炮塔、上层建筑和设备切割。

不能把位于甲板上的炮座或防空炮一并改成木色。优先先改甲板大面，再单独把设备恢复为金属灰。

## 5. 防鱼雷突出部识别

俯视图中的防鱼雷突出部通常是舰体左右两侧的长条对称结构：

- 连通面积较大；
- 宽度远大于高度；
- 靠近舰体外缘；
- 上下两侧形状和位置近似对称。

它们属于金属舰体，不属于木甲板。参考舰通常使用浅灰或中灰，并保留黑色外轮廓和灰度阴影。

## 6. 防空炮识别

博福斯等防空炮通常具有：

- 多组重复的小型圆形或十字形炮座；
- 成对分布在舰体两侧；
- 炮座内可能同时包含黑色线稿、灰色阴影和原涂装色；
- 与木甲板紧邻，但本体应保持金属灰。

修复时逐组框选，并利用连通组件面积、尺寸和对称位置筛选。不要用覆盖整个炮位的矩形把甲板板缝一起染灰。

## 7. 透明背景检查

只删除与画布边缘连通的白底，可能留下被索具、栏杆或桅杆包围的封闭白块。最终候选需要同时在：

- 默认透明预览；
- 黑色背景；
- 品红或棋盘格背景

上检查。封闭白块应使用 `jutland-clean-ship-image` 的组件分析和限定框处理，避免误删白色标线、文字和高光。

## 8. 推荐迭代方式

1. 保存原图不动。
2. 每个正式候选都从原图重新生成；旧候选只用于比较或提供材质掩码。
3. 每轮只改一种材质。
4. 中间版本写入 `/tmp`。
5. 统计改变的像素和包围盒。
6. 放大检查左右对称部位。
7. 发现误染时从修改前候选重新分析，不继续在错误结果上扩大替换规则。
8. 通过人工确认后再进入 `jutland-add-ship`。

## 9. 发虚通常来自线稿被改色

像素舰船图的清晰度主要来自纯黑一像素线稿，而不是分辨率。把黑线改成灰色、连续多轮处理候选图或重新缩放，都会让侧视图显得发虚。

- 最终候选从原始清晰图生成。
- 默认保护所有纯黑像素。
- 如果旧候选发虚，先统计原图纯黑像素有多少被修改。
- 只有确认是实心迷彩填充时，才允许修改纯黑内部；保留其一像素黑边。

## 10. 线稿与黑色迷彩填充的区别

纯黑不能永远无条件保护。舰桥中央可能存在与轮廓线连通的大面积黑色迷彩填充。

- 细线通常填充率低、宽或高只有一至数像素。
- 实心填充在包围盒内的填充率高，内部像素拥有七到八个同色邻居。
- 整组件替换会连带破坏轮廓；只替换高邻居密度的内部像素更安全。
- 处理后保留黑色外边界，并用中灰表达凹部或阴影。

## 11. 清除迷彩不等于压成单一灰色

原迷彩颜色的深浅不能直接继承为新涂装层次，否则同一舱壁会留下斜向或矩形色块。

推荐层次：

| 结构 | 建议灰度 |
|---|---:|
| 主舰体、大面积舱壁 | `193`、`203` |
| 上层塔体和高光平台 | `213`、`219`、`226` |
| 设备侧面、平台底部、内凹阴影 | `174`、`181` |
| 明确深凹部位 | `145`、`164` |

先把连续舱壁统一为主灰，再根据平台底部、内凹位置和设备侧面恢复阴影。灰阶应跟随结构边界，而不是原迷彩边界。

## 12. 二层甲板需要层次，但不能靠黑线堆积

二层甲板可使用接近主甲板的浅蜡黄色。层次来自轻微底色差和较明显但低对比度的暖棕板缝。

- 高密度纯黑板缝在缩小后会合并成大片黑影。
- 可把每两像素一条的板缝降为约每四像素一条。
- 被删除的板缝恢复为相邻木色；保留板缝改成暖棕灰。
- 只修改上下都是木色、左右连续的一像素水平线，避免误伤炮管和设备轮廓。

## 13. 用对称性修复防鱼雷突出部

俯视图的上、下防雷带通常围绕舰体中心线镜像。某一侧误染成木色时，与其按全图 RGB 猜测，不如把另一侧作为材质真值。

1. 确定水平中心轴。
2. 限定误染侧的长条区域。
3. 只选择误染木色或原迷彩色。
4. 只有镜像像素属于允许的金属灰时才复制。
5. 验证目标带内不再残留木色，并保留黑色外轮廓。

## 14. RGBA 编码也需要验证

图片可能所有像素都不透明，但原始文件仍是 RGBA PNG。标准编码器可能把这类图片优化成 RGB，导致 `hasAlpha: no`。

- 比较前后每个像素的 alpha 值。
- 最终使用强制 RGBA 的写出脚本。
- 用 `sips -g hasAlpha` 验证文件编码，而不只依赖解码后的像素值。
