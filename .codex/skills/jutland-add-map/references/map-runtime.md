## 1. 资源链

- `configs/maps.json5`：地图元数据，`name` 对应 `.map` 文件名，`source` 对应 PNG 文件名。
- `resources/maps/<name>.map`：逐字符地形网格；行数是高度，每行字符数是宽度。
- `resources/images/map/abbrs/<source>.png`：缩略图源，也是运行时场景块切片源。
- `pkg/resources/mapcfg/cfg.go`：加载网格、定义 `.`/`O`/`S`/`C`/`L` 和寻路类型。
- `pkg/resources/images/mapblock/block.go`：按网格切源图；`S/C` 有海底图，`L` 没有。
- `pkg/resources/images/abbrmap/abbr_map.go`：合成缩略图底图和地图源图。

图片尺寸必须满足：

```text
PNG 宽度 / 网格宽度 == PNG 高度 / 网格高度 == 单格源像素尺寸
```

当前已注册运行时地图 PNG 均为 4096 像素宽；常用网格为 128 格宽，因此单格源像素尺寸为 32。不要假设这个值：每次都从目标 PNG 和 `.map` 计算，并把 `--cell-pixels` 显式传给生成、验证脚本。运行时再缩放到 `constants.MapBlockSize`。

## 2. 地形字符与绘制

| 字符 | 通航 | 通用海底图 | 源图叠加 | 透明度约束 |
|---|---:|---:|---:|---|
| `.` | 是 | 浅海 | 否 | 全透明 |
| `O` | 是 | 深海 | 否 | 全透明 |
| `S` | 是，代价较高 | 浅海 | 是 | 至少一个可见像素 |
| `C` | 否 | 浅海 | 是 | 至少一个可见像素 |
| `L` | 否 | 否 | 是 | 完全不透明 |

典型方框原因：某海岸格被标成 `L`，源图在水侧存在透明或半透明像素，而运行时不为 `L` 铺海底图。修复地图字符比在 PNG 中硬填蓝色更可靠：水侧改 `S`，阻挡岸线改 `C`。

`.map` 的地形和源图叠加是两件事。不要因为某格“主要透明”就标成 `.`；只要有一个非零 alpha 像素，运行时就需要进入 `S/C/L` 分支才能保留它。

## 3. 矩形地图检查点

首张非正方形地图需要逐项核对：

- `MapData.Get`：分别按行数检查 `y`、按当前行长度检查 `x`。
- 地图初始化：拒绝空图和不等宽行。
- 场景切片：PNG 宽高分别可整除网格宽高，且横纵单格像素相等。
- 缩略图合成：背景和源图分别缩放到目标宽高；不要复用正方形背景的缩放因子裁切源图。
- 任务选择界面：按实际缩略图宽高等比 fit，边框和文字位置使用实际宽度。
- 全屏缩略图：白色相机框、舰船和设施坐标分别除以地图宽高。
- 侧栏缩略图：不要创建固定正方形画布后直接裁切长图。
- 缩略图点击定位：交互区域宽度必须使用实际缩略图宽度，不能把宽高都设为布局高度。

- `pkg/game/draw.go`

- `pkg/mission/drawer/abbr_map.go`
- `pkg/mission/manager/interaction.go`
- `pkg/mission/sidebar/panel.go`

## 4. 珍珠港经验

- 四张 2048×2048 RGBA 原图的编号不代表最终位置；通过边缘误差和地标确认后，行优先顺序为 `pearl1, pearl2, pearl4, pearl3`。
- 四张原图按 2×2 拼成 `4096×4096` 港区；为了得到 `1:1.5` 的运行时范围，在底部增加 2048 透明像素，最终为 `4096×6144`，对应 `128×192` 网格和 32 像素/格。
- 曾经把每张原图先放大到 4096，产生 `8192×10240` 运行资源。这混淆了原始瓦片/中间图和运行时目标尺寸；生成、验证时必须显式声明 `--expected-width 4096 --expected-height 6144 --expected-grid-width 128 --expected-grid-height 192`，让这类错误直接失败。
- “任何非全透明内容都叠加”应实现为：只有 alpha 最大值为 0 的格子才能是 `.`/`O`。
- 仅保证“有内容就叠加”仍不够。半透明格若标成 `L`，水侧会缺海底图并出现阶梯方框；最终还要保证 `partial-land=0`。
- 修复不能只覆盖用户截图岸段。必须扫描整图，确保所有半透明海岸格都是 `S/C`。
- 缩略图只显示左上角且舰船错位，通常是旧正方形缩放逻辑或旧可执行文件仍在运行。
- 地图尺寸扩展不自动授权修改舰队位置；地图资源和关卡设计分开交付。

## 5. 交付检查表

- 原图顺序和缩放方式已记录。
- 原始素材未被覆盖或删除。
- 生成前已明确目标 PNG 宽高、目标网格宽高和 `cell-pixels`；运行时宽度为 4096。
- 生成、验证脚本均传入相同的目标 PNG 和网格宽高，输出的 `declared-target`、`declared-grid` 正确。
- PNG 为 RGBA，尺寸与网格严格对应。
- `.map` 行数、列数和字符集合有效。
- `visible-skipped=0`。
- `partial-land=0`。
- `empty-overlay=0`。
- 整图运行时风格预览无方框和缺口。
- `configs/maps.json5` 引用链完整。
- 非正方形地图的所有缩略图和交互入口已审计。
